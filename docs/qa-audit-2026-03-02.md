## Hallazgos clave

1. **Bloqueo de arranque por inyección de dependencias en módulo de cupones (corregido).**
   - `CouponsController` usa `AdminGuard`, pero `CouponsModule` no declaraba ni importaba los providers necesarios para resolver `JwtService`.
   - Se corrige importando `AuthModule` y registrando `AdminGuard` en `CouponsModule`.

2. **Bloqueo de arranque por variables de entorno obligatorias.**
   - Sin `GOOGLE_CLIENT_ID/GOOGLE_CLIENT_SECRET` el backend falla al construir `GoogleStrategy`.
   - Sin `DATABASE_URL` Prisma no puede inicializarse (`P1012`).

3. **Compilación frontend OK tras instalar dependencias faltantes en Store.**
   - Antes de instalar, `Frontend/Store` fallaba por `Cannot find module 'next-intl/plugin'`.
   - Tras `npm install`, build correcto en ambos frontends.

4. **Limitación de entorno para validación end-to-end real con infraestructura.**
   - `docker` no está disponible en este entorno, por lo que no se pudo levantar Postgres/Redis/RabbitMQ aquí para validar flujo completo con persistencia real.

## Propuestas de mejora e implementación

### Prioridad alta (estabilidad)

- Añadir validación temprana de variables de entorno con esquema (por ejemplo `zod`/`joi`) y mensajes de error claros al boot.
- Crear endpoint de healthcheck compuesto (`/health`) con estado de DB, Redis y RabbitMQ para observabilidad operativa.
- Añadir test e2e mínimo de arranque y rutas públicas con `supertest` + base de datos de pruebas.
- Definir `.env.example` completo para backend, storefront y admin con valores mínimos para levantar en local.

### Prioridad media (calidad de release)

- Pipeline CI con pasos obligatorios:
  - `npm ci`
  - `npm run build`
  - `npm test`
  - e2e smoke tests
- Homogeneizar scripts en frontends (el admin no expone script `lint` actualmente).
- Añadir auditoría de dependencias (`npm audit` controlado, Snyk/GitHub Dependabot).

### Mejoras sugeridas para el panel de administración

- **Gestión de roles y permisos (RBAC)** por recurso/acción (productos, pedidos, cupones, banners, chat).
- **Bitácora de auditoría** de acciones de admin (quién, qué cambió, cuándo, IP/session).
- **Flujos de aprobación** para cambios sensibles (reglas de precio, cupones, featured products).
- **Dashboard operativo** con métricas de negocio y SLA técnico (errores API, tiempos de respuesta, colas pendientes).
- **Herramientas de soporte**: reintentos de sincronización con Infortisa, re-proceso de pedidos fallidos, trazabilidad de incidencias.
- **UX de productividad**: filtros guardados, vistas personalizadas, acciones masivas, import/export CSV.
